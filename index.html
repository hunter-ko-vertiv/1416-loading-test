SP Scalability Phase 2 - collecting ideas and researching tools

Goal

Collect more precise CPU usage for explaining the relationship between users behavior and the status of Firefly

Precondition

At least 25 users where the page is being updated every 15 seconds
Let users randomly fetch different SP data

Experimental method

Feedback require

  • More reliable data collection(grafana + prometheus + node_exporter)
  • Reduce the time to transfer SP to be managed(Spider v.1.2.1 provides speed up)
  • Use K6 replace Robert framework to test
What tools are used for the testing:

  • Grafana - Graphic Integration
  • Prometheus - Monitoring system and time series database
  • Node Exporter - Prometheus plugin
  • K6 - Loading test tool

Grafana

Grafana is the open source analytics & monitoring solution for every database.
Grafana Demo site
How to start
					
						docker run -d --name=grafana -p 3000:3000 grafana/grafana
Account: admin/admin
http://{{host_ip}}:3000

Prometheus

The Prometheus monitoring system and time series database.
How to install prometheus
					
						wget https://github.com/prometheus/prometheus/releases/download/v2.30.0/prometheus-2.30.0.darwin-amd64.tar.gz
						tar -xvf ./prometheus-2.30.0.darwin-amd64.tar.gz
						cd ./prometheus-2.30.0.darwin-amd64
Update prometheus config the launch it
					
						vi prometheus.yml
					
				
					
						scrape_configs:
						- job_name: "node_expoter"
						static_configs:
						- targets: ["localhost:9100"]
					
				
					
						sudo ./prometheus
Check service is working
http://{{host_ip}}:9090

Node Exporter

Exporter for machine metrics.
How to start Node Exporter
					
						wget https://github.com/prometheus/node_exporter/releases/download/v1.2.2/node_exporter-1.2.2.darwin-amd64.tar.gz
						tar -xvf ./node_exporter-1.2.2.darwin-amd64.tar.gz
						cd ./node_exporter-1.2.2.darwin-amd64
					
				
					
						sudo ./node_exporter
Check service is working
http://{{host_ip}}:91000/metrics
